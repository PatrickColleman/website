Firewall: bastion host

 

Een bastion host en de naam die men aan een computer geeft waar de firewallapplicatie op draait.

Het is duidelijk dat deze computer aan een aantal eisen moet voldoen:

  • Men gebruikt een veilige uitvoering van het operating systeem, met de laatste correcties toegepast.
  • Alleen de diensten die nodig zijn, worden geinstalleerd (Telnet, DNS, FTP, SMTP, authenticatie). Er zouden kunnen bugs zitten in andere diensten, maar als die niet geinstalleerd zijn, heeft de machine er geen last van.
  • De proxy is zodanig geconfigureerd dat hij slechts een deel an de bevelen kent en dit omwille van dezelfde reden als in het vorige punt.
  • De proxy heeft eventueel slechts toegang tot een beperkt aantal computers.
  • De proxy moet een goede logging realiseren, zodat men achteraf kan nagaan wat er fout gelopen zou kunnen zijn als er een inbreker aan het werk is geweest.
  • De proxy doet geen schijfoperaties. Daardoor kunnen Trojaanse paarden e.a. moeilijker geimplementeerd worden.
  • Men zal er zeker voor zorgen dat hij geen aanduiding gaat geven van zijn versie van het operating systeem bij het aanloggen. Hackers kunnen daaruit al concluderen welke gaten er voorhanden liggen.
  • Men zal steeds de default user accounts veranderen (niet standaard administrator en paswoord!!!)
  • Bij een dual homed host moet de data steeds door de applicatielaag: disable eventuele routing functionaliteit, anders worden de eventuele application layer functionaliteiten overgeslagen!! (zie verder)
