SSL architectuur

Zoals bovenstaande figuur laat zien, maakt SSL gebruik van TCP voor een betrouwbare, veilige end-to-end service te verkrijgen.

SSL bestaat uit het SSL record protocol en levert beveiligingsdiensten aan hogere lagen, in casu applicaties, zoals http er één is.

Een SSL verbinding is een transport dat een passend type service levert. Deze zijn peer-to-peer relaties. De verbindingen zijn altijd tijdelijk en horen bij één sessie. Er kunnen meerdere veilige verbindingen bestaan tussen twee partijen. Een SSL verbinding (connection) heeft volgende parameters:
- server en cliënt random
-server write MAC secret: geheime sleutel voor MAC operaties bij het zenden op de server
- cliënt write MAC secret: geheime sleutel voor MAC operaties bij het zenden vanop de cliënt.
- server write key: conventionele encryptiesleutel voor communicatie server-client.
- client write key: conventionele encryptiesleutel voor communicatie cliënt-server
- initialisatievectors: initialisatievector voor CBC mode.
- sequence numbers

Een SSL sessie is een associatie tussen een client en een server. Ze worden aangemaakt door het handshake protocol, dat we verder zullen bespreken. Sessies definiëren een verzameling cryptografische beveiligingsparameters die door meerdere verbindingen kunnen worden gebruikt. Sessies worden gebruikt om het dure onderhandelen van nieuwe beveiligingsparameters voor elke verbinding te vermijden.
Een sessie heeft volgende parameters:
- sessie identifier
- peer certificate: X509
- compressiemethode
- encryptie specificaties, zoals welk algoritme gebruikt is, welke hash, ...
- master secret: 48 bytes lang geheim tussen client en server
- resumable: kan de sessie een nieuwe verbinding oprichten?
