Pretty Good Privacy (PGP)
authenticatie

 

Men gaat als volgt te werk:

1. Stel het bericht op.

2. Met SHA-1 maakt men een 160 bit hash code van het bericht

3. Deze hash code wordt versleuteld met RSA, gebruik makend van de private sleutel van de zender

4. De ontvanger gebruikt RSA met de publieke sleutel van de zender.

5. De ontvanger genereert ook een hash code en vergelijkt met de ontvangen hash code. Als ze gelijk zijn, is het bericht authentiek.

 

Er bestaan ook losgekoppelde handtekeningen. Men gebruikt deze wanneer bijvoorbeeld meerdere personen een document moeten ondertekenen. Ze berekenen dan allemaal de handtekening op de oorspronkelijke tekst, in plaats van de tekst plus de reeds berekende handtekeningen.
